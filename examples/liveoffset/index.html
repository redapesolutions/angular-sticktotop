Bacon Ipsum:

Let's assume you are not sure what the size of the next image will be, and have therefore not set it in the CSS. When the below sticky div directive gets created, its offset would be wrong compared to once the image is loaded. So the next sticky div will misbehave

I'm not going to behave well!

You can however use the live-offset attribute to read the offset each time the (throttled) scroll event is triggered.

So the next sticky div will behave as expected (despite the previous one not working)

I however will behave as expected thanks to live offset
